## Overview

This interactive HTML guide walks graduate students and researchers through the complete process of planning, conducting, and reporting a scoping review in the social sciences and humanities. It is designed as a **working companion** — not just a reference — with interactive workspaces where users build their protocol step by step.

The guide follows established scoping review methodology from:

- **PRISMA-ScR** (Tricco et al., 2018)
- **Arksey & O'Malley** (2005) + Levac et al. (2010) enhancements
- **JBI Manual for Evidence Synthesis** — Chapter 11
- **Covidence** protocol development guidance

## Guide Structure

The guide contains **10 steps** organized across **4 phases**:

### Phase 1 — Planning
1. **Welcome** — How the guide works, timeline estimator, save/load instructions
2. **Is a Scoping Review Right?** — Review type comparison, rationale workspace
3. **Research Question (PCC)** — Population-Concept-Context framework with 7 worked SSH examples
4. **Eligibility Criteria** — Inclusion/exclusion, source types, date range, language

### Phase 2 — Searching
5. **Search Strategy** — PCC-to-search-term mapping, database guide, citation managers (Zotero & EndNote), search tracker reference
6. **Screening Sources** — Two-phase screening workflow, Covidence/Rayyan guidance

### Phase 3 — Analysis
7. **Data Charting** — Charting form design, variable planning, piloting
8. **Synthesis & Reporting** — Numerical + thematic synthesis, PRISMA flow diagram, stakeholder consultation

### Phase 4 — Writing
9. **PRISMA-ScR Checklist** — Interactive 22-item tracker with progress bar
10. **Your Protocol Draft** — Compiled protocol, quick-reference card, export options

## Interactive Features

- **Workspaces on every step** — Text fields where users draft each protocol section as they go. All entries are preserved in session and compile into a full protocol on Step 10.
- **Auto-carry-forward** — PCC terms entered in Step 3 automatically populate as starting search terms in Step 5, reducing redundant typing.
- **PCC question generator** — Enter Population, Concept, and Context to auto-generate a draft research question with color-coded PCC tags.
- **Clickable framework examples** — 7 worked PCC examples across SSH disciplines (Education, Social Work, Sociology, Public Policy, LIS, Psychology/Migration, Environmental Humanities). Clicking any example auto-fills the workspace.
- **Boolean search string preview** — Combines search terms into a formatted AND/OR preview.
- **Source type pills** — Toggleable selection chips for source types (peer-reviewed, grey lit, theses, policy docs, etc.).
- **PRISMA-ScR checklist** — 22 clickable items with a progress bar.
- **Common pitfalls per step** — Collapsible "⚠️ Common Pitfalls" sections on each step with 3–5 contextual warnings.
- **Timeline estimator** — Adjustable inputs for team size, databases, and expected results volume produce an estimated phase-by-phase timeline.
- **Save/Load to file** — Save all workspace entries to a `.json` file and reload them later across sessions and devices.
- **Quick-reference card** — Auto-populated one-page summary of key decisions on the final step, designed for printing.
- **Protocol compiler** — Compiles all entries into a formatted protocol draft with copy-to-clipboard, print, and search string examples with database-specific adaptation notes.

## Companion Spreadsheet

**`scoping_review_search_tracker.xlsx`** includes four worksheets:

1. **Search Log** — Database name, date, exact search string, number of results, export status
2. **Citation Tracking** — Import and deduplication counts by source with auto-calculated totals
3. **Full-Text Exclusions** — Exclusion reasons log for PRISMA flow diagram reporting
4. **PRISMA Flow Numbers** — Auto-calculated counts for identification, screening, eligibility, and inclusion stages

## How to Use

1. **Open `scoping_review_guide.html`** in any modern browser (Chrome, Firefox, Safari, Edge). No internet connection required after initial load.
2. **Navigate** using the top bar (Prev/Next buttons, dropdown selector, or arrow keys).
3. **Fill in workspaces** as you progress. Entries are preserved as you navigate between steps.
4. **Save progress** using the "Save to File" button at the top. This downloads a `.json` file you can reload later.
5. **Compile your protocol** on the final step. Copy to clipboard or print.
6. **Use the search tracker** (`scoping_review_search_tracker.xlsx`) to document your searches as you run them.

## Technical Notes

- **Single HTML file** — No external dependencies, frameworks, or build tools. Works offline.
- **No server required** — Runs entirely in the browser. No data is transmitted anywhere.
- **Save files are plain JSON** — Can be inspected, edited, or version-controlled.
- **Print-optimized** — CSS print stylesheet hides navigation and shows all steps for clean printing.
- **Keyboard navigation** — Left/Right arrow keys move between steps (when not focused on a text field).
- **Mobile responsive** — Adapts to smaller screens, though the desktop experience is recommended for workspace-heavy steps.
